# 1. **프로젝트 개요**

### 🧭 목적

**디자인·개발 전공자**가

- 프로젝트
- 포트폴리오
- 트렌드 리서치
- 학습 성장 관리

이 모든 것을 *한 곳에서* 관리할 수 있는 **전공자 통합 플랫폼** 구축.

### 🧩 핵심 가치

- 매일 쓰고 싶게 만드는 "즉시 가치 기능" 제공
- 포트폴리오·프로젝트 **수동 업로드 기반 정리**
- 실력 측정 → 맞춤형 프로젝트 템플릿 → 성장 추적까지 일관된 흐름 구축
- 트렌드·스킬·작업 기반 자동 분석 제공

### 슬로건

**"Your Design & Dev Growth, All in One."**

# 2. **배경 및 전략**

## 2-1. 전공자 Pain Point (리서치 기반)

- 시간표가 중요한 시기는 1~2주, **그 외 학기 대부분은 프로젝트·포폴 관리가 핵심**
- Notion / GitHub / Figma / Google Drive / 캠퍼스 커뮤니티를 각각 따로 사용
- "내 실력이 어느 수준인지" 알기 어려움
- 졸업작품·방학 프로젝트·포폴 방향에 대한 기준 부족
- 트렌드 탐색이 분산됨 (Awwwards / Mobbin / Dev.to / Medium / GitHub 등)
- **포트폴리오를 만들려고 해도 "뭘 만들어야 할지" 알 수 없음**

👉 **시간표 없이도 매일 방문할 이유가 필요하며, 구체적인 프로젝트 방향성이 필수.**

---

## 2-2. 경쟁사 분석

| 서비스 | 강점 | 약점 | 본 서비스의 차별점 |
| --- | --- | --- | --- |
| **에브리타임** | 대학 커뮤니티·확산력 | 전공 관련 기능 없음 | 전공자 중심 자동화 기능 |
| **노션(Notion)** | 생산성/정리 최강 | 자동화·전공자 향 기능 없음 | 포폴 진단 + 맞춤 템플릿 + 성장 로드맵 |
| **포트폴리오 메이커** | 빠른 템플릿 | 분석·버전관리 없음 | 실력 분석 + 성장 로드맵 |
| **프로그래머스/코드잇** | 스킬 학습 | 포트폴리오·학기 관리 없음 | 작업 기반 실력 진단 + 프로젝트 템플릿 |
| **Mobbin/Awwwards** | UI 트렌드 | 개인화 안 됨 | 전공자 맞춤 트렌드 추천 |

### 결론

**"뭘 배워야 하나?" + "뭘 만들어야 하나?"를 동시에 해결하는 경쟁자 부재.**

# 3. **타깃 사용자**

### 🎨 김서연 (21, 디자인과 2학년)

- 매 학기 UI/UX 과제와 포폴 정리를 반복
- 트렌드 탐색과 실력 향상이 필요
- "이 정도 수준이면 어떤 프로젝트를 만들어야 포트폴리오에 좋을까?" 고민

### 💻 박준형 (23, 컴공 3학년)

- React·GitHub·세미프로젝트 경험 많음
- 자신의 실력 수준을 객관적으로 알고 싶어함
- "다음 프로젝트 주제가 뭘까?" 매번 고민

### 👨‍🎓 이지은 (24, 졸업 예정)

- 포트폴리오를 이제 완성해야 하는 상황
- 기존 작업물들을 정리하고 버전관리를 하려는 상태
- 취업 전 최종 점검이 필요

# 4. **핵심 기능 (7가지 초기 유입 훅)**

## 🟦 **① 기술·디자인 트렌드 10초 카드**

**목적:** 빠른 흥미 유발 & 첫 방문 증가

**동작:**

- Awwwards / Mobbin / Dev.to / GitHub Trending / Medium 등에서 일일 큐레이션
- 카드 형태로 제시 (이미지 + 제목 + 한 줄 설명)
- 사용자가 반응(좋아요/저장/패스) → 관심 태그 자동 수집

**연결:**

- 관심 태그 → **⑥ 개인화 콘텐츠 추천** 정확도 상승
- 저장된 트렌드 → 프로젝트 템플릿 영감 제공

---

## 🟦 **② 전공자 스킬 레벨 10초 테스트**

**목적:** 실력 확인 욕구 충족 & 초기 온보딩 강화

**동작:**

- 각 영역(디자인/프론트엔드/풀스택)별 10개 객관식 문제
- 10초 안에 완료 가능한 빠른 형식
- 결과 제시: "상위 30% 수준" / "부족한 항목: React 상태관리"

**측정 기준:**

- **디자인:** 기본기·일관성·인터랙션·그리드 (Superside 기반)
- **프론트엔드:** HTML/CSS/JS 문법·컴포넌트·상태관리 (Index.dev 기반)
- **풀스택:** 위 + DB·API·배포 (RocketDevs 기반)

**결과 카드:**

```
┌─────────────────────────────┐
│ 당신의 레벨: Senior Beginner │
│                             │
│ 강점:                       │
│ ✓ CSS/Tailwind (상위 40%)   │
│                             │
│ 개선 필요:                   │
│ ✗ React State (하위 60%)    │
│                             │
│ → 맞춤 프로젝트 템플릿 보기   │
└─────────────────────────────┘

```

**연결:**

- 결과 → **③ 맞춤형 프로젝트 템플릿** 추천
- 재테스트 → **⑦ 포폴 성장 그래프** 업데이트

---

## 🟦 **③ 맞춤형 프로젝트 템플릿 + 로드맵**

**목적:** "뭘 만들어야 할지" 구체적 방향 제시 & 초반 몰입 유도

**동작:**

### Phase 1: 실력별 프로젝트 템플릿 제공

사용자 레벨(②의 결과)에 따라 **4주 단위 프로젝트 로드맵** 자동 생성.

### 📋 **레벨 1 (입문): "UI 기초 마스터"**

- **1주:** 디자인 시스템 따라하기 (Figma 템플릿 제공)
- **2주:** 반응형 랜딩페이지 (HTML/CSS)
- **3주:** 간단한 Vue/React 앱 (ToDo, 날씨 앱)
- **4주:** 포트폴리오 정리 & 버전 v1 생성

**템플릿 포함:**

- 피그마 스타트 파일
- 코드 스켈레톤 (GitHub 링크)
- 매주 체크리스트
- 완료 후 평가 루브릭

---

### 📋 **레벨 2 (초급): "풀 프로덕트 만들기"**

- **1주:** 디자인 시스템 확장 (컴포넌트 라이브러리)
- **2주:** React + Tailwind 상태관리 실습
- **3주:** 미니 서비스 구현 (영화 검색/날씨 대시보드)
- **4주:** 백엔드 연결 기초 (REST API)

**템플릿 포함:**

- Next.js 스타터 보일러플레이트
- Figma Design System 파일
- API 문서 + Postman 컬렉션
- 배포 가이드 (Vercel)

---

### 📋 **레벨 3 (중급): "풀스택 포트폴리오 프로젝트"**

- **1주:** 데이터베이스 설계 (Figma → ERD)
- **2주:** 백엔드 API 개발 (Node/Spring)
- **3주:** 프론트엔드 + 백엔드 통합
- **4주:** 배포 + 성능 최적화

**템플릿 포함:**

- 백엔드 스타터 (Spring Boot / Node.js)
- 풀스택 프로젝트 예제
- Docker 설정
- CI/CD 가이드

---

### Phase 2: 템플릿 활용 추적

- 템플릿 선택 → 프로젝트 보드(⑤)로 자동 연결
- 매주 체크리스트 상태 추적
- 완료 시 **포폴 업로드**(사용자 수동 업로드) 유도

**연결:**

- 완료 프로젝트 → **④ 주간 성장 리포트** 포함
- 완료 프로젝트 → **⑤ 포폴 아카이브** 저장 권유
- 업로드된 포폴 → **⑦ 성장 그래프** 반영

## 🟦 **④ 주간 성장 리포트 자동 생성**

**목적:** 활동 요약 & 주간 재방문 유도

**동작:**

- 매주 일요일 자동 생성
- 내용: 트렌드 소비량 / 레벨 테스트 결과 / 프로젝트 진행률 / 포폴 업로드 현황

**리포트 형태:**

```
┌──────────────────────────────────────┐
│ 이주의 성장: 당신은 32시간 성장했어요 │
│                                      │
│ 📊 활동 요약                         │
│ · 트렌드 카드 본 개수: 42개          │
│ · 레벨 테스트: 1회 (React 상향)     │
│ · 프로젝트 진행: 40% → 60%          │
│ · 포폴 업로드: 0개 (이번주)         │
│                                      │
│ 🎯 다음주 권유 활동                  │
│ · 프로젝트 완료 후 포폴 업로드       │
│ · React State 심화 학습 추천         │
│                                      │
│ 📱 공유하기 (SNS)                    │
└──────────────────────────────────────┘

```

**SNS 공유 효과:**

- 친구 초대 → 신규 사용자 자연 유입
- SNS에 공유된 리포트 → 바이럴 유도

**연결:**

- 리포트 → 부족 항목 명시 → **③ 새 템플릿** 추천
- 리포트 → **⑦ 성장 그래프**에 누적

**추가 인사이트:**

- 기술스택 다양성 (사용한 언어/프레임워크)
- 프로젝트 복잡도 추이
- 디자인/개발 비중
- 팀/개인 프로젝트 비중

```
① 트렌드 카드
   ↓ (관심 태그 수집)
   → ⑥ 개인화 콘텐츠 정확도 상향
   → ③ 프로젝트 템플릿 영감

② 레벨 테스트
   ↓ (실력 측정)
   → ③ 맞춤형 템플릿 추천
   → ⑦ 성장 그래프 기준점

③ 프로젝트 템플릿
   ↓ (구체적 방향 제시)
   → ⑤ 프로젝트 보드에 추가
   → ④ 주간 리포트 진행률 추적

④ 주간 성장 리포트
   ↓ (활동 요약 + SNS 공유)
   → 신규 사용자 자연 유입
   → 기존 사용자 재방문 유도

⑤ 프로젝트 아카이브
   ↓ (완료 프로젝트 정리)
   → ⑦ 포폴 업로드 권유
   → ④ 리포트에 진행률 반영

⑥ 개인화 추천
   ↓ (매일 3개 콘텐츠)
   → Daily Habit 형성
   → DAU(일일활성사용자) 증가

⑦ 포폴 버전관리 + 성장그래프
   ↓ (장기 기록 + 시각화)
   → 졸업작품/취준 핵심 자산
   → 장기 유지(LTV) 최강 기능
   → ② 레벨 테스트 재진단 기준

```

# 5. **포트폴리오 업로드 시스템 (상세)**

### 💡 왜 "자동 생성"이 아닌 "수동 업로드"인가?

1. **품질 관리:** 사용자 스스로 작품을 정리하는 과정에서 성찰 발생
2. **실제 경험:** 진정한 포트폴리오 제작 경험 제공
3. **기술 스택 분석 정확성:** 자동으로 판단하는 것보다 사용자 입력이 정확함
4. **신뢰도:** "자동 생성 포폴"보다 "직접 만든 포폴"이 취업 시장에서 신뢰도 높음

### 📋 업로드 플로우

```
Step 1. 프로젝트 보드(⑤)에서 완료된 프로젝트 선택
        ↓
Step 2. "포폴에 추가" 버튼 클릭
        ↓
Step 3. 포폴 정보 입력 (프로젝트명, 설명, 기술스택 등)
        ↓
Step 4. 스크린샷/영상 업로드
        ↓
Step 5. 자동으로 카드 생성 & 포폴 보드 저장
        ↓
Step 6. 버전관리 시스템에 기록
        ↓
Step 7. ⑦ 성장 그래프 자동 업데이트

```

### 🔧 입력 필드 (최소화 설계)

**필수 입력:**

- 프로젝트명
- 한 줄 설명 (50자)
- 기술스택 (선택식)
- 스크린샷 (최소 1장)

**선택 입력:**

- 긴 설명 (마크다운)
- GitHub 링크
- 배포 링크
- 역할 (Design/Frontend/Backend/Fullstack)

**자동 추가:**

- 업로드 날짜
- 카테고리 (기술스택 기반 자동 분류)
- 썸네일 자동 생성

# 6. **프로젝트 템플릿 상세 (③번 심화)**

### 📚 템플릿 구성 원칙

1. **레벨별 난이도 조정**
    - Level 1 (입문): 기초 개념 이해, 간단한 구현
    - Level 2 (초급): 실전 기능 구현, 상태관리
    - Level 3 (중급): 풀스택 프로젝트, 배포/성능
2. **4주 단위 마이크로 학습**
    - 주 1개의 구체적 학습 목표
    - 매주 체크리스트로 진행률 추적
    - 완료 시 포폴 업로드 자동 권유
3. **포폴 자산화 연결**
    - 각 템플릿 완료 후 포폴 업로드 유도
    - 버전관리로 성장 기록
    - 취준/졸작 시점에 강력한 포트폴리오 확보

### 📖 Level 1 샘플: "UI 기초 마스터"


# 8. **수익화 전략 (SaaS 구독 모델 - 추후 적용)**

### 현재 (Phase 1: Free)

- ✅ 모든 기능 무료 제공
- ✅ 기본 포폴 저장 무제한
- ✅ 7가지 핵심 기능 모두 무료

### 향후 (Phase 2: Freemium → SaaS)

### **Free Tier (영구 무료)**

- ① 트렌드 카드: 무제한
- ② 레벨 테스트: 무제한 (월 1회 결과 저장)
- ④ 주간 리포트: 생성만 (공유 X)
- ⑤ 프로젝트 보드: 최대 5개
- ⑥ 개인화 추천: 제한적
- ⑦ 포폴 카드: 최대 5개

### **Pro Tier (₩4,900/월)**

- ③ 프로젝트 템플릿: 무제한 다운로드
- ② 레벨 테스트: 월 무제한 결과 저장 + 상세 리포트
- ④ 주간 리포트: SNS 공유 + 고급 통계
- ⑤ 프로젝트 보드: 무제한
- ⑦ 포폴 카드: 무제한 + 공개 포폴 도메인 (예: portfolio.username.app)
- 추가: 포폴 분석 AI (Superside 기반)

### **Team Tier (₩9,900/월)**

- Pro 모든 기능
- 팀 협업 기능 (복수 사용자 포폴 관리)
- 팀 리포트 (팀원 성장 추적)
- 우선 지원 (이메일)

### **Enterprise (Custom)**

- 대학 커스텀 패키지
- 학과별 대시보드
- API 접근
- 전담 지원
